# 部署SPA项目，刷新非首页页面报404错误

这是一个典型的 Nginx 单页应用（SPA）路由配置问题。当你在浏览器中直接访问或刷新非首页路由时，Nginx 会尝试在服务器上查找对应的文件或目录，但实际这些路由是由前端 JavaScript 处理的。

## 解决方法

```nginx
server {
    # ... 其他配置 ...
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # 如果你的项目在子目录，比如 /admin/
    # location /admin/ {
    #     try_files $uri $uri/ /admin/index.html;
    # }
}
```
