# 工作流程和流程

## Web项目中的典型技术组合

构建网站时，你会用到多种不同的技术组合，通常称为 技术栈。

Web项目通常分为前端和后端，并通过API进行通信。

### 前端

负责用户能看到并与之交互的部分。

1. 核心三语言：HTML（结构）、CSS（样式）、JavaScript（逻辑）。
2. 前端框架/库：用于构建复杂的、交互式的单页面应用。
    - React： 由Facebook开发，目前最流行的库，生态庞大。
    - Vue.js： 渐进式框架，易于上手，学习曲线平缓。
    - Angular： 由Google开发的完整框架，功能全面但学习曲线较陡。
3. 状态管理：用于管理复杂的应用级状态。
    - Redux、Mobx (常与React搭配)， Pinia、Vuex (与Vue搭配)。
4. 构建工具：提升开发效率和代码质量。
    - Vite： 新一代构建工具，速度极快。
    - Webpack： 老牌且功能强大的模块打包器。
5. CSS框架：快速实现美观、响应式的布局。
    - Tailwind CSS： 实用优先的原子化CSS框架。
    - Bootstrap： 老牌成熟的UI组件库。
6. UI组件库
    - Ant Design： 企业级 React UI 库，设计语言和组件质量非常高，是国内React生态的绝对主流。
    - Material-UI (MUI)： 实现了 Google Material Design 的 React 组件库，非常流行。
    - Element Plus： 一套为 Vue 3 准备的优质桌面端 UI 组件库。
    - Vuetify： 为 Vue.js 打造的 Material Design 框架。
7. 移动端框架
    - 原生iOS： 使用 Swift 或 Objective-C，配合 Xcode。
    - 原生Android： 使用 Kotlin 或 Java，配合 Android Studio。
    - React Native：编译原生流派。使用 JavaScript 编写业务逻辑，通过“桥接”与原生模块（如相机、GPS）通信。UI 组件会被翻译成对应的原生组件（如 View -> UIView）。
    - NativeScript：与 RN 类似，但可以直接调用所有原生 API，且不依赖 React，可以选择 Angular, Vue 或原生 JavaScript 进行开发。
    - Flutter：自渲染引擎流派。使用 Skia 图形引擎直接绘制 UI，完全绕过原生组件。它不使用 WebView，也不使用平台的原生控件。
    - uniapp：WebView 流派（混合应用）。应用本质上是一个内嵌的浏览器，UI 是网页。通过插件调用原生设备功能。

### 后端

负责服务器、应用逻辑、数据库和认证等，是网站的“大脑”。

1. 后端语言与框架：
    - Node.js + Express/Nest.js： 使用JavaScript进行全栈开发。
    - Python + Django/Flask： Django功能全面（“全家桶”），Flask轻量灵活。
    - Java + Spring Boot： 大型企业级应用的首选，稳健强大。
    - C# + ASP.NET Core： 微软技术栈，性能优异。
    - PHP + Laravel/Symfony： 在Web开发中历史悠久，生态成熟。
    - Go + Gin： 以高性能和高并发著称，适合API和微服务。
2. 数据库：
    - 关系型数据库： MySQL， PostgreSQL（更高级，支持JSON等）。
    - 非关系型数据库： MongoDB（文档型）， Redis（键值对，用作缓存）。
3. API设计：RESTful API 是目前最主流的设计风格，GraphQL 是一种更灵活、高效的新兴选择。

###  部署与运维

1. 云服务： AWS， Google Cloud Platform， Microsoft Azure， 阿里云。
2. 容器化： Docker 将应用及其依赖打包成镜像。
3. 容器编排： Kubernetes 用于自动化管理大量Docker容器。
4. Serverless： 无需管理服务器，按需执行代码（如 AWS Lambda）。

## 网站开发团队的典型工作角色

| 角色 | 核心职责 |
| -- | -- |
| 产品经理 | 定义产品愿景、规划路线图、撰写需求文档、协调各方资源。 |
| UI/UX 设计师 | UX：设计用户流程、线框图、信息架构，关注易用性。UI：负责视觉设计，如配色、字体、图标，产出高保真设计稿。 |
| 前端开发者 | 将设计师的设计稿转化为可交互的网页，负责用户体验和性能优化。 |
| 后端开发者 | 编写服务器逻辑、数据库设计、API接口，确保数据安全与处理效率。 |
| 全栈开发者 | 兼顾前端和后端开发，能够独立完成功能模块。 |
| DevOps 工程师 | 负责自动化部署、监控、维护服务器和网络基础设施，提升开发与运维效率。 |
| QA（质量保障）工程师 | 编写测试计划、执行手动和自动化测试（功能、性能、安全），提交Bug。 |
| 项目经理 | 负责项目进度、资源分配、风险控制、团队协调，确保项目按时交付。 |

## 典型的技术项目阶段，以及其中涉及的不同工作角色

一个项目通常遵循以下阶段，不同开发流程（如下文的敏捷）会迭代这些阶段。

| 阶段 | 描述 | 主要参与角色 |
| -- | -- | -- |
| 1. 需求分析与规划 | 明确项目目标、目标用户、功能范围、技术可行性及预算时间。 | 产品经理、项目经理、客户/利益相关者、技术主管 |
| 2. 设计 | UI/UX设计： 创建线框图和视觉稿。技术设计： 设计系统架构、数据库 schema、API 接口。 | UI/UX设计师、架构师、后端/前端主管 |
| 3. 开发与实现 | 前端、后端开发者根据设计稿和技术方案编写代码，并行工作并通过API集成。 | 前端开发者、后端开发者、全栈开发者 |
| 4. 测试与QA | 对已完成的功能进行系统测试，包括功能测试、集成测试、性能测试和安全测试。 | QA工程师、所有开发者（负责单元测试） |
| 5. 部署与上线 | 将代码部署到生产环境，让最终用户能够访问。 | DevOps工程师、后端开发者、项目经理 |
| 6. 运维与迭代 | 监控系统运行状态，修复线上问题，收集用户反馈，规划并开发下一个版本。 | DevOps工程师、全体开发团队、产品经理 |

## 常见的工作管理流程，例如敏捷开发和瀑布式开发

### 瀑布式开发

一种传统的、线性的方法。

- 流程： 需求分析 → 设计 → 实现 → 测试 → 部署 → 维护。必须完成前一阶段，才能进入下一阶段。
- 优点： 流程清晰，文档完备，易于管理。
- 缺点： 不灵活，后期变更需求代价巨大，客户直到最后才能看到产品。
- 适用场景： 需求非常明确、固定且不会变更的项目，如大型政府工程、银行核心系统。

### 敏捷开发

一种迭代式、增量式的开发方法，旨在快速响应变化。

- 核心价值： 个体和互动 高于流程和工具，可工作的软件 高于详尽的文档，客户合作 高于合同谈判，响应变化 高于遵循计划。
- Scrum（最流行的敏捷框架）：
    - 角色： 产品负责人、Scrum Master、开发团队。
    - 工件：
        - 产品待办列表： 所有需要完成的功能列表。
        - 冲刺待办列表： 当前迭代周期（Sprint）计划完成的任务。
    - 仪式：
        - 冲刺规划会： 规划本周期要做什么。
        - 每日站会： 15分钟同步进度、困难和计划。
        - 冲刺评审会： 向产品负责人演示已完成的功能。
        - 冲刺回顾会： 团队反思如何改进流程。
- 看板：
    - 可视化工作流（如：待办、进行中、完成），限制在制品数量，强调持续交付。
- 优点： 高度灵活，能快速响应市场和客户需求的变化，客户参与度高。
- 缺点： 范围可能不断变化，对团队成员的自律和协作能力要求高。

### 混合模式

在实践中，很多团队会结合瀑布和敏捷的优点。例如，在宏观层面使用瀑布式进行规划和设计，在具体开发阶段使用敏捷框架进行迭代冲刺。
