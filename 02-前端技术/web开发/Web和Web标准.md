# Web和Web标准

## Web

Web，全称万维网，是一个由无数互相关联的文档和其他资源组成的全球性信息系统，通过互联网访问。

核心要点：

1. Web ≠ 互联网

- 互联网：是底层的基础设施，一个全球性的计算机网络，负责连接设备（如TCP/IP协议）。
- Web：是建立在互联网之上的一个服务或应用。它使用互联网来传输数据，就像电子邮件、在线游戏一样，只是互联网的众多应用之一。

2. Web 的核心技术
- URL：统一资源定位符，即资源的“地址”。
- HTTP/HTTPS：超文本传输（安全）协议，规定了浏览器和服务器之间如何通信。
- HTML：超文本标记语言，用于构建网页的内容和结构。

3. 核心理念：超文本

- Web 的本质在于“链接”。通过超链接，用户可以非线性地从一个文档跳转到另一个相关的文档，形成一个巨大的“信息网”。

### Web 简史

1. 序幕：前 Web 时代 (1945-1989)

- 范内瓦·布什 提出“Memex”设备构想，被视为超文本的灵感来源。
- 泰德·纳尔逊 创造“超文本”一词。
- 互联网（TCP/IP 协议）本身已经诞生，主要用于学术和军事机构，但缺乏一个简单、统一的信息访问系统。

2. Web 1.0 时代：只读的静态网页 (1989-2004)

核心特征： 信息展示，用户是消费者，单向传播。

| 时间 | 里程碑事件 | 意义 |
| -- | -- | -- |
| 1989 | 蒂姆·伯纳斯-李 在欧洲核子研究组织撰写了《关于信息管理的提案》。 | 万维网的诞生。他提出了三大核心技术：HTML、URL 和 HTTP。 |
| 1991 | 第一个网站 info.cern.ch 上线，解释了万维网是什么。 | 世界进入了“网站”时代。 |
| 1993 | Mosaic 浏览器发布。 | 第一个流行的图形化浏览器。它让普通用户也能轻松浏览网页，极大地推动了 Web 的普及。 |
| 1994 | 蒂姆·伯纳斯-李创立 万维网联盟。网景通信公司成立。 | 开始致力于制定统一的 Web 标准，避免碎片化。“浏览器大战”初现端倪。 |
| 1995 | 网景 Navigator 与 微软 Internet Explorer 的“浏览器大战”爆发。 | 商业竞争白热化，虽然带来了快速创新，但也导致了糟糕的兼容性问题（如 IE 特有的技术）。 |
| 1990s 末 | 电子商务（Amazon，eBay）、搜索引擎（Yahoo!，Google）出现。 | Web 开始展示其商业潜力，dot-com 泡沫开始积聚。 |
| 2000 | Dot-com 泡沫破裂。 | 大量互联网公司倒闭，市场回归理性，为下一次健康发展奠定了基础。 |

3. Web 2.0 时代：可读可写的社交网络 (2004-2010s)

核心特征： 用户生成内容，社交互动，平台崛起。用户是参与者。

- “Web 2.0”一词在2004年流行起来。
- 关键技术： AJAX（如 Google Maps）使得网页无需刷新即可更新内容，用户体验得到质的飞跃。
- 代表性产品/现象：
    - 博客 & Wikipedia：每个人都可以成为发布者。
    - 社交媒体：YouTube, Facebook, Twitter, 微博。内容创造和社交成为核心。
    - 共享经济：Uber, Airbnb。
- 商业模式： 基于数据和注意力的“免费”模式——用户免费使用服务，平台通过广告和用户数据盈利。
- 影响： 催生了“平台经济”，但也带来了数据隐私、虚假信息和垄断等问题。

4. Web 3.0 时代：可读-可写-可拥有的语义网络/去中心化网络 (2010s - 现在)

核心特征： 数据互联、智能、去中心化。用户是所有者。

Web 3.0 的定义尚未完全统一，但通常包含以下几个层面：

1. 语义网络（最初的 Web 3.0 愿景）
- 目标： 让机器能够理解和处理数据本身的意义。
- 比喻： 从“文档的网络”进化为“数据的网络”。
- 技术： RDF, OWL, 知识图谱。
> 例子： 当你搜索“爱因斯坦的生日”，搜索引擎不是找到一个包含这句话的文档，而是直接告诉你“1879年3月14日”，因为它理解了你的问题和数据的含义。

2. 去中心化网络（当前 Web3 的主流叙事）
- 核心思想： 打破 Web 2.0 时代科技巨头的垄断，将权力和数据归还给用户。
- 基石技术： 区块链、加密货币、智能合约、NFT。
- 核心特征：
    - 无需信任： 依赖代码和数学，而非中心化公司。
    - 所有权： 用户真正拥有自己的数字资产（如 NFT）和数据。
    - 抗审查： 数据分布在网络中，难以被单一实体删除或控制。
> 例子： <br>
> DeFi：去中心化金融。 <br>
> NFT：数字艺术和物品的所有权证明。 <br>
> dApps：建立在区块链上的去中心化应用。 <br>

3. 空间网络 / 元宇宙
- 目标： 将 Web 变成一个融合物理和数字世界的、沉浸式的三维空间。
- 技术： VR, AR, 物联网。
- 例子： Meta 的元宇宙愿景，微软的 Mesh。

5. 可能的未来：Web 4.0？ (展望)

虽然尚无定论，但一些趋势已经开始显现：

- symbiotic Web：人与机器深度融合、协同思考。大脑-计算机接口可能成为新的交互方式。
- 无处不在的智能：人工智能深度融合到 Web 的每一个角落，提供高度个性化的、主动的服务。
- 更极致的沉浸感：元宇宙技术成熟，虚拟与现实的边界更加模糊。

## Web标准

Web 标准是一系列由标准化组织制定的技术规范和指南，其目的是确保 Web 的长期增长和根本稳定性。

### 为什么需要 Web 标准？

想象一下，如果没有交通规则，所有汽车厂商自己决定方向盘在哪边、信号灯颜色代表什么，那将是怎样的混乱。Web 在早期就经历了这种“浏览器战争”的混乱。

Web 标准的目标是：

- 互操作性：确保任何遵循标准的浏览器都能正确显示任何遵循标准的网站。
- 可访问性：确保网站能被所有人访问，包括残障人士（如使用屏幕阅读器的视障用户）。
- 长期可维护性：使网站代码结构清晰，易于开发和维护。
- 设备兼容性：让网站能适应从台式机到手机、平板等各种设备和屏幕尺寸。

### 谁制定 Web 标准？

主要的标准化组织是：

- W3C：万维网联盟，由 Web 发明人蒂姆·伯纳斯-李创立，是 Web 标准最主要的制定者。
- WHATWG：Web 超文本应用技术工作组，由各大浏览器厂商组成，主要负责 HTML 和 DOM 等核心标准的演进。
- ECMA International：负责标准化 ECMAScript（即 JavaScript 语言的核心）。

### 核心的 Web 标准有哪些？

| 标准 | 全称与作用 | 简单解释 |
| -- | -- | -- |
| HTML | 超文本标记语言 | 网站的骨架和内容。它用各种标签（如 `<h1>, <p>, <img>`）来定义网页的结构和含义（语义）。 |
| CSS | 层叠样式表 | 网站的外观和样式。它控制网页的布局、颜色、字体、动画等，负责“美化”工作。 |
| JavaScript | （ECMAScript 标准） | 网站的交互行为。它使网页能够响应用户操作（如点击按钮、提交表单）、动态更新内容等。 |

除了这三大核心，Web 标准还包括：

- 可访问性标准：如 WCAG，为开发者提供指南，确保网站内容对残障人士可访问。
- Web 性能指南：提供如何让网站加载更快、响应更迅速的实践标准。
- 语义 Web 标准：如 RDF, 微格式 等，旨在让网络数据对机器更友好、更易理解。

## Web 与 Web 标准的关系

Web 标准是确保 Web 健康、开放和普惠的基石。

1. 开放与公平：标准是公开、免费的。任何人、任何公司都可以依据标准开发浏览器或创建网站，避免了技术垄断，确保了 Web 的开放性。
2. 一致性体验：因为有标准，用户无论使用 Chrome、Firefox、Safari 还是 Edge，都能获得基本一致的浏览体验。
3. 推动创新：标准提供了一个稳定、统一的基础平台，开发者可以在此基础上放心地构建复杂、创新的应用，而无需担心底层兼容性问题。
4. 面向未来：标准化的代码更“坚固”，不易被未来的浏览器淘汰，保护了数字资产的投资。

## 作为开发者/设计师，为什么要遵循 Web 标准？

1. 降低开发成本：写一套代码，就能在多个浏览器上运行，无需为不同浏览器编写特定版本。
2. 提升网站性能：遵循标准的代码通常更简洁、高效，加载速度更快。
3. 改善 SEO：搜索引擎更喜欢结构良好、符合标准的网站，这有助于提高搜索排名。
4. 增强可访问性：让你的网站服务于更广泛的用户群体，这不仅是道德要求，在很多地区也是法律要求。
5. 便于维护与协作：结构清晰、符合标准的代码更容易被你自己或团队其他成员理解和修改。
