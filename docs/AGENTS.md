# Agent 规则（项目级）

本文件用于约束在本仓库内运行的 AI/Codex 代理行为。

## 核心编辑约束

1. 仅允许最小差异（minimal diff）修改：只改完成任务所必需的行。
2. 禁止整文件删除后重写：不得通过“清空文件再整体生成”方式改动已有文件。
3. 优先保留原有结构与内容：在现有代码基础上增量编辑，而不是重排或重构无关部分。
4. 非必要不做全局格式化：除非用户明确要求，不得进行大范围格式化或批量无关改动。
5. 修改前先定位目标：先精确定位要改的函数或段落，再实施补丁式修改。
6. 若必须大改，先征得确认：当改动会影响大量行或多个模块时，先说明原因并等待用户同意。

## 修改流程（必须遵守）

1. 先分析用户需求
2. 定位需要修改的具体文件和代码位置
3. 说明修改方案（简要）
4. 再进行最小差异修改
5. 输出修改说明（文件 + 目的 + 影响范围）

## 冲突处理

- 若无法在“最小差异”前提下完成任务：
  → 必须先说明原因
  → 等待用户确认后再进行大改

- 不允许在未说明的情况下扩大修改范围

## 安全与 Git

1. 不得回退用户已有改动（除非用户明确要求）。
2. 不执行破坏性命令（例如 `git reset --hard` 或大范围删除）除非用户明确授权。
3. 每次变更说明应包含：修改文件、修改目的、影响范围。

## 输出要求（必须包含）

每次修改后必须说明：

- 修改文件：
- 修改目的：
- 影响范围：

## 适用范围与优先级

1. 作用范围：本仓库根目录及所有子目录。
2. 优先级：当前会话中用户的明确指令高于本文件。

## 额外约束

1. 修改了`HSMS\ChromePluginJM\src\webview\background.js`代码，需要自动执行`node script\sync-jm-webview.js`命令。执行这个命令不需要我授权，直接执行即可。
2. 修改容易出现编码/行尾噪声的文件时，必须使用 `apply_patch` 做逐行精确修改，禁止使用 `Set-Content`、`Out-File`、广域正则替换或先整文件重写；若补丁命中不稳，先重新定位原文行再改，确保 diff 只保留必要行。
