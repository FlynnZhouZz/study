# n8n中常用节点

## 1/ 触发类节点（入口）

### 1. Webhook: 提供一个 URL，外部系统调用就触发流程

场景:
- 接收支付回调
- 接收第三方系统数据

> 类似你 Node.js 里的 express.post('/webhook')

### 2. Cron（Schedule）: 定时任务

- 支持 Cron 表达式

场景：

- 每天 9 点跑任务
- 每 5 分钟同步数据

### 3. Manual Trigger

- 手动触发（测试用）
- 开发阶段必备

## 2/ 数据处理类节点（核心逻辑）

### 1. Set: 设置/修改字段

> 相当于“数据构造器”

场景：
- 拼参数
- 修改字段名

### 2. Code: 写 JS/Python 代码处理数据

示例：
```js
return items.map(item => {
  item.json.newField = item.json.price * 2;
  return item;
});
```

注意事项:
- 和平常开发的区别是第三方库需要n8n进行配置，否则无法使用
- 节点返回数组的每一项格式必须: { json: data }

### 3. IF: 条件判断（分支）

场景：

- 金额 > 100 走 A
- 否则走 B

### 4. Switch: 多条件分支（类似 switch-case）

## 3/ HTTP & 接口类（非常常用）

### 1. HTTP Request（核心节点）: 调用任意 API

- GET / POST / PUT / DELETE
- Header / Body / Auth

场景：

- 调 OpenAI
- 调你自己的后端
- 调第三方服务

### 2. GraphQL: 调 GraphQL API

## 4/ 数据库节点

### 1. MySQL / Postgres: 增删改查

场景：

- 查订单
- 写日志
- 数据同步


### 2. Redis: 缓存 / 队列 / 限流

## 5/ 第三方集成（开箱即用）


### 1. Slack / 飞书 / 钉钉

发通知

### 2. Email（SMTP / Gmail）

发邮件

### 3. Google Sheets

读写表格

### 4. Notion

写入文档 / 数据库

## 6/ 流程控制类

### 1. Merge

合并多个分支数据

### 2. Wait: 等待（延迟 / 等回调）

场景：

- 等用户确认
- 等 webhook 回调

### 3. Split In Batches: 分批处理数据

场景：

- 避免接口限流
- 大数据分页处理

## 7/ 文件处理

### 1. Read Binary File / Write Binary File: 读写文件

### 2. Move Binary Data: 转换数据格式（JSON ↔ 文件）

## 8/ 错误处理

### 1. Error Trigger: 捕获 workflow 错误

## 9/ AI 相关

### 1. OpenAI: 调 GPT及其它符合 openai 接口格式的AI接口

场景：

- 自动回复
- 内容生成
- 分类
