# 快速入门

## 如何开始使用？

1. 体验云端版：直接在 [n8n.io](https://n8n.io) 注册一个云账户，可以快速上手，无需任何部署工作 。
2. 自行部署：如果您希望完全掌控，可以通过 npm (npx n8n) 在本地启动，或者使用 Docker 进行部署，都非常简单。

## 使用npm本地安装n8n

### 1. 环境配置

1. 安装node.js: 推荐使用nvm版本管理工具安装nodejs。>=20.19 && <= 24.x
2. 安装python: 推荐使用pyenv (pyenv-win)。>=3
3. 安装Visual Studio（windows环境需要）：[Visual Studio官网下载](https://visualstudio.microsoft.com/zh-hans/downloads)

### 2. 全局安装 n8n
```bash
npm install n8n -g
```

### 3. 启动 n8n
安装完成后，直接在终端里输入 n8n 命令即可启动：
```bash
# 检查版本
n8n --version

# 启动测试
n8n start

# 启动
n8n
```

你会看到终端输出 `http://localhost:5678` 的地址

### 4. 访问并完成设置

打开浏览器，访问 `http://localhost:5678`。首次访问时，按照页面提示注册一个管理员账号，之后就可以进入 n8n 的主界面了

特别说明：
> 这种直接通过 n8n 命令启动的方式，一旦关闭终端窗口，n8n 就会停止运行。它非常适合临时测试或开发。如果你希望 n8n 能像服务一样长期在后台运行，可以考虑使用 pm2 等进程管理工具，或者选择官方推荐的 Docker 方式进行部署

## windows 环境安装 常见失败

### 文件权限不足（导致EPERM清理错误）

解决方法：以管理员身份重新运行

许多 EPERM 错误是因为npm没有权限删除或修改受系统保护的文件。

1. 关闭所有命令行窗口（CMD、PowerShell 或 Git Bash）。
2. 找到你的命令行程序，右键点击，选择 “以管理员身份运行”。
3. 在管理员终端中，先尝试清理npm缓存：
```bash
npm cache clean --force
```
4. 重新执行安装命令
```bash
npm install n8n -g
```

### 缺乏必要的C++编译环境（导致sqlite3安装失败）
```bash
gyp ERR! find VS You need to install the latest version of Visual Studio including the "Desktop development with C++" workload.
```

安装必要的构建工具

1. 下载生成工具：访问官方下载页面：[Visual Studio](https://visualstudio.microsoft.com/zh-hans/downloads/#build-tools-for-visual-studio-2022) 
2. 打开Visual Studio Installer：在Windows开始菜单搜索"Visual Studio Installer"并打开
3. 修改安装：找到你已安装的Visual Studio（如2022 Community） - 点击右下角的 "修改" 按钮
4. 添加工作负载：
- 在工作负载标签页中，找到并勾选 "使用C++的桌面开发"
- 右侧会默认勾选必要的组件（MSVC编译器、Windows SDK等）
- 点击右下角的 "修改" 开始安装
- 等待下载和安装完成（大约需要2-3GB空间）

配置npm使用正确的VS版本:
```bash
# 设置msvs版本为2022
npm config set msvs_version 2026

# 设置Python路径（虽然你已经有了，但确保node-gyp能找到）
npm config set python "C:\Users\Admin\.pyenv\pyenv-win\versions\3.12.4\python3.exe"
```

### 清理并重新安装n8n
```bash
# 1. 卸载现有n8n
npm uninstall -g n8n

# 2. 清理缓存
npm cache clean --force

# 3. 手动删除残留文件夹（可选）
# 打开文件资源管理器，删除 D:\software\nvm\v22.22.1\node_modules\n8n

# 4. 重新安装
npm install -g n8n
```
