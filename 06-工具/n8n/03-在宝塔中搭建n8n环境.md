# 在宝塔中搭建n8n环境

最推荐的方法是使用 Docker，因为这样最快、最干净，也最方便后续的升级和维护。

## 1. 安装 Docker 环境

宝塔面板：

- ≤3.9.3：在宝塔的“软件商店”中找到并安装“Docker管理器
- ≥3.9.3：在宝塔面板的侧导航 Docker

## 2. 一键部署 n8n

进入Docker板块，在应用商店中搜索“n8n”并直接点击安装，只需简单配置即可:

- 域名：需解析域名到对应服务器
- 端口：默认或者自定义
- cpu核心数限制：默认或者自定义
- 内存限制：默认或者自定义

端口需宝塔面板和服务器安全组都要放行指定端口。

## 3. 编辑工作流或者导入已编写好的工作流json文件

- 访问绑定的n8n docker服务域名
- 进入n8n工作台
- 创建或者导入

### 注意事项：

- 工作流中的webhook是localhost没正确的，因为宝塔有做nginx代理，配置的域名会默认绑定到n8n服务中

## 可能问题

### 0、宝塔面板安装 n8n 容器失败

需要将宝塔面板更新到最新版本才行

### 1、工作流的 http request节点有请求本服务器的服务（接口）

这种情况下，直接写服务的域名地址是会报错的:

> DNS server returned an error, perhaps the server is offline

原因:

> 当你在 n8n 容器内部请求一个域名（如 klynode.158178.cloud）时，容器会尝试通过 DNS 解析该域名。如果这个域名指向的是服务器的 公网 IP，由于某些云服务商的防火墙限制或 NAT 回流（Hairpin NAT）问题，容器可能无法从内部绕回到宿主机的公网接口，从而导致 DNS 报错或连接超时。

解决方案:

- 宝塔面板 - docker - 容器编排 - 找到对应的容器 - 进入容器 编排页面
- 编辑配置文件(docker-compose)

```yml
services:
    n8n:
        image: n8nio/n8n:${VERSION}
        #    container_name: ${CONTAINER_NAME}
        deploy:
            resources:
                limits:
                    cpus: ${CPUS}
                    memory: ${MEMORY_LIMIT}
        restart: always
        ports:
            - ${HOST_IP}:${WEB_HTTP_PORT}:5678
        volumes:
            - ${APP_PATH}/data:/home/node/.n8n
        extra_hosts:
            - "klynode.178.cloud:172.17.0.1" # 重点
        environment:
            N8N_SECURE_COOKIE: false
            N8N_PROTOCOL: https
            WEBHOOK_URL: https://${N8N_HOST}
        labels:
            createdBy: "bt_apps"
        networks:
            - baota_net

networks:
    baota_net:
        external: true
```

### 2、报错 `Module 'crypto' is disallowed [line 1]`

在节点 code - javascript 中，使用了 `const crypto = require("crypto");`。

原因:

> 因为 n8n 出于安全考虑，默认禁止在 Code 节点中调用 Node.js 的内置模块或第三方库

方案一：使用node原生 crypto

```yml
environment:
    NODE_FUNCTION_ALLOW_BUILTIN: crypto
```

方案二：使用 n8n 内置 $crypto

```js
const hash = $crypto.sha256("hello");
return [{ hash }];
// 或者
const hmac = $crypto.hmac("sha256", "data", "secret");
return [{ hmac }];
```

保存 - 重启docker

### 3、http request 请求外部域名，dns解析超时

如：请求 1688接口返回
`getaddrinfo EAI_AGAIN h5api.m.1688.com`错误。

说明你的 n8n 容器完全无法解析外部域名。

EAI_AGAIN 是 DNS 查询超时或解析器无响应的典型报错。对于 1688 这种国内域名，如果你的服务器在国内，通常是 Docker 的网桥（Bridge）与宿主机的 DNS 配置脱节了。

解决方法：

- 宝塔面板 - Docker - 设置 - 配置寄文件

```yml
{
    "registry-mirrors": ["https://docker.1ms.run"],
    "dns": ["223.5.5.5", "114.114.114.114", "8.8.8.8"],
}
```

添加上 dns 节点

- "223.5.5.5": 阿里 DNS
- "114.114.114.114": 114 DNS
- "8.8.8.8": Google DNS
